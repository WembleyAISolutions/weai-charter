# WEAI Charter v1.0

## Constitutional Sovereignty Document

------------------------------------------------------------------------

## 1. Purpose

The WEAI Charter defines the constitutional layer of the WEAI ecosystem.
It establishes structural sovereignty, non-bypassable governance
constraints, and canonical authority over all foundational components.

All WEAI components MUST comply with this Charter.

------------------------------------------------------------------------

## 2. Constitutional Layers

WEAI ecosystem is structured into five immutable layers:

1.  Charter (Sovereignty Layer)
2.  Canon (Normative Rules)
3.  Kernel (Deterministic Execution Engine)
4.  Runtime (Commercial & Policy Enforcement)
5.  Registry (Skill & Schema Authority)

Layer dependency direction is strictly top-down. No lower layer may
override or mutate a higher layer.

------------------------------------------------------------------------

## 3. Structural Sovereignty Principles

### 3.1 Non-Bypassability

No component may execute outside the Kernel when state mutation occurs.

### 3.2 Deterministic Integrity

All state transitions must be deterministic, replayable, and
fingerprint-verifiable.

### 3.3 Immutability of Execution History

Execution logs and ledger entries are append-only.

### 3.4 Separation of Meaning and Structure

Kernel enforces structure only. Runtime enforces policy and commercial
logic. Business meaning is never embedded in Kernel.

------------------------------------------------------------------------

## 4. Authority Hierarchy

Charter \> Canon \> Kernel \> Runtime \> Registry

If conflict occurs: - Higher layer prevails. - Lower layer must adapt.

------------------------------------------------------------------------

## 5. Governance Enforcement

Mandatory controls:

-   Replay validation enforcement
-   Fingerprint consistency checks
-   Engine-only mutation rule
-   Version progression enforcement
-   Adapter isolation boundaries

Failure to comply constitutes structural violation.

------------------------------------------------------------------------

## 6. Versioning Doctrine

Charter versions are immutable once released. Major revisions require
formal version increment. No retroactive mutation permitted.

------------------------------------------------------------------------

## 7. Scope

This Charter governs:

-   WEAI Kernel
-   WEAI Runtime
-   WEAI Registry
-   All platform implementations
-   All skill execution frameworks

------------------------------------------------------------------------

## 8. Declaration

WEAI is a structural execution sovereignty system. It is not a business
platform. It is not a pricing engine. It is not a UI framework.

It is the constitutional execution layer of agent-based economic
systems.

------------------------------------------------------------------------

END OF DOCUMENT
