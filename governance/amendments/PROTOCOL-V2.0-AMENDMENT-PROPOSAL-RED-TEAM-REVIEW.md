## 7. New Governance Gaps Identified by This Review

This review identifies two additional governance gaps that should be considered before any future Protocol v2.0 body is drafted.

### New Gap 1 — Review artifact linkage to proposal lifecycle state

The current repository now contains a proposal record and a review artifact pattern.

Future governance procedure should define how review artifacts are linked to proposal lifecycle state without implying adoption.

Required future decision:

```text
Does a repository-recorded review artifact change proposal state automatically, or must proposal-state promotion require a separate explicit governance record?
```

### New Gap 2 — Proposal state update after review

The reviewed proposal states that it is a review candidate before repository-recorded review.

After a review artifact exists, future procedure should define whether and how the proposal state may be updated.

Required future decision:

```text
Should proposal state be updated in the proposal file, in a separate review index, in the review artifact, or only in a later adoption record?
```
